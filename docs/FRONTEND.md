# Frontend Guide

프론트엔드가 있는 프로젝트에서 UI 작업의 기준을 기록합니다.

## UI Principles

- 첫 화면은 사용자가 실제 작업을 시작할 수 있어야 합니다.
- 로딩, 빈 상태, 오류 상태를 반드시 처리합니다.
- 비동기 흐름은 로딩, 성공, 오류 상태가 코드에서 분리되어 보여야 합니다.
- 화면 안의 텍스트와 버튼은 모바일과 데스크톱에서 겹치지 않아야 합니다.
- 복잡한 계산과 데이터 변환은 렌더링 코드 밖으로 분리합니다.
- 파생 상태를 중복 저장하지 않고, 상태 이름은 화면 의미를 드러내게 둡니다.

## Component Rules

| 요소 | 원칙 |
|---|---|
| Container | 데이터 로딩, 상태 전환, 이벤트 연결 |
| Presentational | 표시와 사용자 입력만 담당 |
| Form | 입력 검증과 오류 메시지를 명확히 표시 |
| Modal | 닫기, 취소, 확인 흐름을 분명히 제공 |

## Verification

- 주요 화면의 정상, 로딩, 빈 상태, 오류 상태를 확인합니다.
- 반응형 화면에서 텍스트 겹침과 레이아웃 깨짐을 확인합니다.
