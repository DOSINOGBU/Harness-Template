# Dependencies

새 라이브러리는 코드 한 줄이지만 이후 운영에 큰 영향을 줍니다.
AI 에이전트는 의존성을 추가하기 전에 이 문서를 확인합니다.

## When To Add

| 조건 | 통과 |
|---|---|
| 표준 라이브러리/현재 의존성으로 해결 불가 | yes |
| 직접 구현 시 신뢰성·보안 리스크가 큼 | yes |
| 비슷한 기능의 기존 의존성이 이미 있음 | no, 기존 것 사용 |
| 한두 함수만 필요함 | no, 직접 구현 검토 |

## Evaluation Checklist

- [ ] 라이선스가 프로젝트와 호환됩니다.
- [ ] 최근 1~2년 내 유지보수 이력이 있습니다.
- [ ] 다운로드 수, 이슈 흐름, 보안 권고를 확인했습니다.
- [ ] 번들 크기 또는 시작 시간 영향이 작습니다.
- [ ] 동등한 기능의 대안 1~2개를 비교했습니다.
- [ ] 단일 관리자 의존, 미러링 안 되는 패키지가 아닙니다.

## Add Procedure

1. 위 체크리스트를 통과시킵니다.
2. 사용자 승인을 받습니다.
3. 잠금 파일(lockfile)을 함께 커밋합니다.
4. 사용 위치를 한 곳으로 모아 도입 범위를 추적합니다.
5. ADR이 필요한 큰 결정이면 `docs/adr/`에 기록합니다.

## Remove Procedure

- 사용처를 모두 찾아 제거합니다.
- 잠금 파일과 설정 파일에서 함께 제거합니다.
- 빌드, 테스트, 런타임 시작이 정상인지 확인합니다.
- 관련 환경변수, 설정값을 함께 정리합니다.

## Update Procedure

| 변경 폭 | 검증 |
|---|---|
| Patch | 빌드와 단위 테스트 |
| Minor | 빌드, 테스트, 핵심 흐름 수동 검증 |
| Major | 변경 노트 확인, 영향 범위 조사, 마이그레이션 계획 |
