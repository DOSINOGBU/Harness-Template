# Security

보안 관련 변경을 할 때 AI 에이전트가 반드시 확인해야 하는 기준입니다.

## Sensitive Data

- API 키, 토큰, 비밀번호, 개인식별정보를 코드나 로그에 남기지 않습니다.
- `.env.example`에는 값이 아니라 변수명과 설명만 둡니다.
- 사용자 입력은 신뢰하지 않고 경계에서 검증합니다.

## Permission Rules

| 작업 | 원칙 |
|---|---|
| 인증 | 실패 원인을 과도하게 노출하지 않음 |
| 권한 확인 | 서버 측에서 다시 검증 |
| 파일 업로드 | 확장자, 크기, 콘텐츠 타입 검증 |
| 외부 요청 | 허용된 목적과 도메인만 사용 |

## Security Review Checklist

- 민감정보가 로그에 남지 않는가?
- 사용자 입력이 검증되는가?
- 권한 검사가 클라이언트에만 있지 않은가?
- 에러 메시지가 내부 구조를 과도하게 드러내지 않는가?
- 외부 입력이 그대로 SQL, 셸, HTML, 경로에 들어가지 않는가?
- 새 의존성이 보안 권고를 받은 적이 없는가?

## Threat Model Notes

| 자산 | 위협 | 1차 방어 |
|---|---|---|
| 사용자 자격 증명 | 탈취, 재사용 | 안전한 저장, 전송 구간 암호화 |
| 개인정보 | 무단 조회, 유출 | 접근 권한 검사, 감사 로그 |
| 외부 비밀키 | 노출, 폐기 누락 | 환경변수, 롤링 정책 |
| 신뢰 경계 입력 | 주입, 위변조 | 경계 검증, 출력 인코딩 |
