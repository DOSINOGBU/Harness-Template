# Agent Behavior

AI 에이전트가 코드를 작성하거나 수정할 때 기본으로 지킬 행동 기준입니다.
상세한 프로젝트 규칙은 각 도메인 문서를 따르고, 이 문서는 과잉 구현과 추측 수정을 막는 짧은 기준으로 사용합니다.

## Think Before Coding

- 확실하지 않은 요구사항은 조용히 가정하지 않습니다.
- 해석이 여러 가지면 가능한 해석과 tradeoff를 먼저 드러냅니다.
- 더 단순한 접근이 있으면 구현 전에 말합니다.
- 혼란이 남아 있으면 이름 붙여 설명하고 확인합니다.

## Simplicity First

- 요청받지 않은 기능을 추가하지 않습니다.
- 한 번만 쓰는 코드에 추상화를 만들지 않습니다.
- 미래 확장성을 이유로 복잡도를 미리 넣지 않습니다.
- 확인되지 않은 시나리오를 위한 에러 처리나 설정을 과하게 만들지 않습니다.

## Surgical Changes

- 필요한 파일과 라인만 변경합니다.
- 주변 코드, 주석, 포맷을 함께 개선하지 않습니다.
- 기존 스타일이 마음에 들지 않아도 현재 작업에서는 맞춥니다.
- 내가 만든 미사용 코드만 정리하고, 기존 dead code는 발견사항으로 남깁니다.
- 모든 변경 라인은 사용자 요청이나 검증 필요성과 연결되어야 합니다.

## Goal-Driven Execution

- 작업을 검증 가능한 성공 기준으로 바꿉니다.
- 버그 수정은 가능하면 실패를 먼저 재현한 뒤 고칩니다.
- 리팩터링은 변경 전후 동작이 같음을 확인합니다.
- 여러 단계 작업은 각 단계마다 확인 방법을 둡니다.

## Learning Loop

- 한 번 발생한 실수는 `docs/LEARNING_LOG.md`에 메모합니다.
- 두 번 반복되면 관련 checklist에 반영합니다.
- 세 번 반복되면 `docs/AUTOMATION_EVOLUTION.md`에 자동화 후보로 올립니다.
