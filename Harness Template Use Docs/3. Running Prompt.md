# Running Prompt

아래 프롬프트는 프로젝트용 하네스 초기화가 끝난 뒤, 실제 구현을 시작할 때 사용합니다.

한 번에 전체 앱을 만들게 하지 않고, `docs/exec-plans/active/`에 있는 작업 중 하나만 구현하게 하는 것이 핵심입니다.

```text
이제 `docs/exec-plans/active/`에 있는 MVP 작업 중 가장 우선순위가 높은 것부터 하나만 구현해줘.

규칙:
- 작업 전 관련 문서를 읽어줘.
- 한 번에 하나의 기능만 구현해줘.
- 큰 파일이 생기지 않게 책임을 분리해줘.
- 로딩/빈/오류 상태를 포함해줘.
- 구현 후 테스트/검증을 실행하고 결과를 보고해줘.
- 커밋은 아직 하지 말고 변경 요약과 검증 결과만 알려줘.
```

## 반복 사용 방식

1. AI가 구현할 작업 단위를 하나 고릅니다.
2. 사람이 작업 범위가 적절한지 확인합니다.
3. AI가 구현하고 검증합니다.
4. 사람이 동작, UX, 범위 초과 여부를 확인합니다.
5. 문제가 없으면 커밋/PR 단계로 넘어갑니다.

## 추가로 붙이면 좋은 조건

필요한 경우 아래 조건을 프롬프트 끝에 추가합니다.

```text
추가 조건:
- 새 파일이 필요하면 책임 단위별로 분리해줘.
- 500줄 이상 파일에 새 기능을 추가해야 한다면 먼저 분리 계획을 보고해줘.
- 외부 API 호출에는 실패 로그와 사용자에게 보여줄 오류 상태를 포함해줘.
- 테스트가 불가능한 부분은 수동 검증 방법을 명확히 적어줘.
```
