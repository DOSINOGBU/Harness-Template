# Prompt: Debug Issue

```text
이 문제를 바로 고치지 말고 먼저 분석해줘.

절차:
1. 문제를 한 문장으로 다시 적어줘.
2. 재현 조건을 명확하게 정리해줘. (입력, 환경, 순서)
3. 가능한 원인을 3~5개로 나누고 각 원인의 근거를 적어줘.
4. 가장 가능성 높은 원인을 코드와 로그로 확인해줘.
5. 확인된 원인에 한해 최소 수정안을 제안해줘.
6. 수정 후 재현 시나리오로 다시 검증할 방법을 적어줘.

출력 형식:
1. 🔍 Problem Summary
2. ⚙️ Possible Causes
3. 🎯 Root Cause
4. 🛠 Fix (minimal diff)
5. ✅ Validation Steps

참고:
- 추측으로 수정하지 마.
- 같은 증상이 두 가지 원인을 가질 수 있으니 단정하지 마.
- 임시 해결과 근본 해결을 분리해서 보여줘.
- 확인되지 않은 원인으로 전체 구조를 다시 쓰지 마.
```
