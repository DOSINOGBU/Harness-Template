# Prompt: Prepare PR

```text
현재 변경으로 PR 초안을 준비해줘.

절차:
1. 변경 파일과 diff를 확인하고 한 PR로 묶어도 되는지 판단해줘.
2. 여러 목적이 섞여 있으면 PR 초안을 만들지 말고 분리 기준을 제안해줘.
3. `docs/PULL_REQUESTS.md`와 `.harness/checklists/pull-request.md` 기준으로 PR 생성 조건을 확인해줘.
4. `docs/QUALITY_SCORE.md`의 최소 통과 기준을 PR 본문에 요약해줘.
5. 검증이 부족하면 부족한 검증과 대체 확인 방법을 먼저 보고해줘.
6. 문제가 없으면 `[type] summary` 형식의 PR 제목 후보와 PR 본문 초안을 작성해줘.

규칙:
- 실행 불가능한 상태면 PR 준비 완료로 보고하지 마.
- 테스트 실패, 빌드 실패, 타입 에러, 충돌이 있으면 merge 가능하다고 말하지 마.
- 기능 추가와 리팩터링처럼 목적이 다른 변경이 섞여 있으면 분리해줘.
- 민감정보가 포함되어 있으면 PR 초안을 만들지 마.
- UI 변경이 있으면 스크린샷 필요 여부를 본문에 표시해줘.
- PR 본문에는 검증, 리스크, Quality Score 체크를 포함해줘.
```
