# Checklist: Pull Request

- [ ] PR 목적이 하나로 명확합니다.
- [ ] 리뷰 가능한 크기로 변경을 분리했습니다.
- [ ] 관련 없는 리팩터링, 포맷, 문구 변경을 제거했습니다.
- [ ] 앱 실행, 테스트, 린트, 타입체크, 빌드 또는 수동 검증 결과를 기록했습니다.
- [ ] 실행하지 못한 검증과 대체 확인 방법을 기록했습니다.
- [ ] 영향받는 화면, 컴포넌트, 상태, API를 적었습니다.
- [ ] 리스크와 주의할 영역을 적었습니다.
- [ ] UI 변경이 있으면 스크린샷 필요 여부를 확인했습니다.
- [ ] 토큰, API 키, 계정 정보 같은 민감정보가 포함되지 않았습니다.
- [ ] merge 금지 조건에 해당하지 않습니다.
