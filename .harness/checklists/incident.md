# Checklist: Incident Response

- [ ] 영향 범위(사용자, 기능, 시간)를 한 문장으로 적었습니다.
- [ ] 증상의 시작 시각과 최근 변경을 비교했습니다.
- [ ] 임시 완화책으로 영향을 멈췄습니다.
- [ ] 원본 로그, 메트릭, 재현 조건을 보존했습니다.
- [ ] 임시 완화와 근본 원인 수정을 별도로 분리했습니다.
- [ ] 사용자/이해관계자에게 현재 상태를 공유했습니다.
- [ ] 사후 기록(원인, 영향, 재발 방지)을 남겼습니다.
- [ ] 재발 방지를 코드, 테스트, 문서, 체크리스트 중 어디에 반영할지 결정했습니다.
